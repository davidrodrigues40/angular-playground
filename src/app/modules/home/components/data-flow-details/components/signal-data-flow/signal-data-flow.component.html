Signals Data Flow

Using Angular signals coupled with a service which exposes the state so I don't have to worry about binding the effect and which service to call for each piece of data.

Each component will inject the service associated with the state it needs. The service will expose actions as events and the signal as observables. It will also allow to bind to an effect under the effects property.
If a component requires data from multple states then that component will inject multiple services.

  • The service's effects will bind objects to signals.
  • The service's events will trigger.perform various functions needed to update the signal including calling other services.
  • The service's observables with return the signal's current value.
