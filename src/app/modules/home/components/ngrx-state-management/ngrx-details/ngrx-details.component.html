NGRX State Management

For full documentation on NGRX click here.

This section will talk about how I use NGRX in this application.
I break it down to the following 5 parts:

State This is where the information is stored for a given feature.
Actions These are the actions that can be performed on the state.
Effects I like to think of these as event handlers for performing actions on state data.

These are injectable classes which handle the actions that require more than just a simple state update. When the action is complete they will trigger the appropriate action for the reducer to handle.
Reducers I like to think of these as event handlers for updating the state.

These objects handle the actions which change the state.
Selectors These are functions which return a memoized selector containing a subset of the state.
